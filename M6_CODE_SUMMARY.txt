================================================================================
ARCHITECTURE HIGHLIGHTS
================================================================================

SUBSYSTEM INTEGRATION:
- Both managers are GameInstanceSubsystems
- Initialize during UGameInstance::Init()
- Tick via game mode or explicit tick calls
- Delegates broadcast to UI/gameplay code

ACTIVITY FLOW:
1. StartActivity() creates activity state
2. Setup methods (SetupCheckpointSprint, etc.) find/configure actors
3. TickActivity() updates timers and checks conditions
4. Callbacks (OnCheckpointReached, OnDeliveryPickedUp) update progression
5. CompleteActivity/FailActivity() end activity and record results
6. ClearActivityState() cleans up delegates and actors

CHECKPOINT SPRINT:
1. All checkpoints found and sorted by index
2. First checkpoint marked as reachable
3. Player reaches checkpoint → next becomes reachable
4. Final checkpoint marked with bIsFinish → triggers completion

DELIVERY RUN:
1. All pickup/dropoff points found
2. Pickups enabled, dropoffs disabled initially
3. Pickup entered → cargo picked up, dropoffs enabled
4. Dropoff entered → activity completes
5. Time limit checked throughout

TRAFFIC WEAVE:
1. Simple counter (IncrementTrafficVehiclesPassed)
2. Called by vehicle system when pass detected
3. Time limit runs in background
4. Activity remains active until completion externally

DAMAGE SURVIVAL:
1. UpdateAccumulatedDamage() called by vehicle system
2. If exceeds MaxDamageAllowed → auto-fails
3. Time limit runs in background
4. Activity remains active until time runs out

SAVE/LOAD:
1. UpdatePlaytime() accumulates time
2. At 60-second interval, CaptureGameState() and save
3. LoadGame() captures then ApplyGameState()
4. BestTimes persists across sessions
